# .cursor 행동 검증 시나리오 체크리스트

> 저자: jimmy
> 날짜: 2026-02-12

LLM 기반 에이전트/스킬/모드 행동은 결정적(deterministic) 테스트가 불가능합니다.
이 문서는 수동으로 검증할 수 있는 시나리오를 정의합니다.

각 시나리오를 실행한 후 결과를 기록하고, 기대와 다른 행동이 있으면 해당 룰/스킬/에이전트를 수정합니다.

## 2. 스킬 활성화 시나리오

적절한 상황에서 올바른 스킬이 활성화되는지 확인합니다.

### S-08: 코드 작성 시 code-accuracy 활성화

- 검증: 커밋 메시지가 Conventional Commits 형식

  - clean-functions 스킬 원칙 적용

### S-11: 문서화 요청 시 documentation 활성화

- 기대 행동:

## 3. 훅 트리거 시나리오

hooks.json에 정의된 훅이 의도된 시점에 실행되는지 확인합니다.

### S-13: 세션 시작 시 setup-check (manifest 없음)

- 조건: manifest.json 미생성 상태
- 기대 행동:
  - 세션 시작 시 "manifest.json not found. Run /setup" 메시지 표시
- 검증: 첫 응답에 부트스트랩 안내 포함
- 결과: [ ]

### S-14: 세션 시작 시 setup-check (Ralph Loop 중단)

- 조건: task-folder 내 ralph-state.md에 `active: true` 상태
- 기대 행동:
  - 세션 시작 시 "Task '{task-folder}' interrupted" 메시지 표시
- 검증: 첫 응답에 해당 task-folder명과 Ralph Loop 재개 안내 포함
- 결과: [ ]

### S-15: 위험 명령 실행 시 guard-shell 차단

- 입력: 에이전트에게 "git push --force origin main 실행해줘"라고 요청
- 기대 행동:
  - guard-shell 훅이 deny 판정
  - 에이전트가 차단 사유를 사용자에게 전달
- 검증: force push가 실행되지 않고 안전한 대안 제시
- 결과: [ ]

### S-16: 편집 실패 시 recovery 메시지

- 조건: StrReplace가 old_string을 찾지 못해 실패
- 기대 행동:
  - recovery 훅이 "Re-read the file" 메시지 제공
  - 에이전트가 파일을 다시 읽고 재시도
- 검증: 에이전트가 자동으로 복구 시도
- 결과: [ ]

### S-17: 파일 편집 후 comment-checker 경고

- 조건: 불필요한 주석이 많은 코드 파일 편집
- 기대 행동:
  - comment-checker 훅이 의심 주석 개수 경고
  - comment-checker 스킬 실행 권장
- 검증: 경고 메시지에 주석 유형별 개수 포함

## 5. 4-Phase 워크플로우 시나리오

- 입력: "이 변수명을 userId에서 userIdentifier로 바꿔줘" (단일 파일)
- 기대 행동:
  - Subagent 위임 없이 직접 수정

- 검증: 빠르게 직접 수정, 과도한 분석 없음
- 결과: [ ]

### S-23: 중간 작업 → Execute + Verify

- 입력: "API 엔드포인트에 페이지네이션 파라미터를 추가해줘" (2-3 파일)
- 기대 행동:
  - implementer로 구현 위임
  - reviewer로 검증 (또는 직접 검증)
  - Phase 0(분석), Phase 1(기획) 생략 가능

## 검증 실행 가이드

### 검증 주기
- 스킬/에이전트/훅 추가/수정 시
- evolve 스킬로 .cursor 진화 후
- 주요 Cursor 업데이트 후

### 검증 방법
1. 새 세션을 열고 시나리오의 "입력"을 실행합니다
2. "기대 행동"과 실제 행동을 비교합니다
3. "검증" 항목을 확인하고 결과를 기록합니다
4. 불일치가 있으면 해당 룰/스킬/에이전트를 수정합니다
