# 온보딩 가이드

> 저자: jimmy | 날짜: 2026-02-16

## cursor_symbiote를 처음 사용하나요?

이 가이드는 cursor_symbiote를 처음 접하는 개발자가 5분 안에 핵심을 이해하고 사용을 시작할 수 있도록 도와줍니다.

## 한 문장 요약

cursor_symbiote는 Cursor IDE의 AI Agent를 전문가 팀(분석가, 기획자, 비평가, 구현자, 디버거, 리뷰어)처럼 운영하면서, 사용 패턴에 따라 설정이 자동으로 진화하는 시스템입니다.

## 시작하기

### 1. 설치 (1분)

```bash
# .cursor/ 폴더를 프로젝트 루트에 복사
chmod +x .cursor/hooks/*.sh
```

### 2. 초기화 (2분)

Cursor 채팅에서:

```
/setup
```

코드베이스를 자동 분석하여 프로젝트 스택을 감지하고 설정을 생성합니다.

### 3. 사용 시작 (바로)

그냥 자연어로 요청하면 됩니다:

```
"로그인 화면을 만들어줘"
```

Synapse 오케스트레이터가 작업 복잡도를 판단하고 적절한 에이전트를 자동으로 조율합니다.

## 알아야 할 5가지 핵심 개념

### 1. Synapse 오케스트레이터

모든 요청을 받아 적절한 전문가 에이전트에게 위임합니다. 단순한 질문은 직접 처리하고, 복잡한 작업은 4-Phase Workflow(분석→기획→구현→검증)를 사용합니다.

### 2. 자율 실행 모드

"끝까지 완성해줘"라고 말하면 Ralph Mode가 활성화되어 사용자 개입 없이 Plan→Execute→Verify를 반복합니다. `/autopilot`은 병렬로 더 빠르게 처리합니다.

### 3. 안전장치 (Hooks)

직접 신경 쓰지 않아도 자동으로 동작합니다:
- 위험한 Shell 명령은 실행 전에 차단됩니다
- 불필요한 AI 주석이 자동으로 감지됩니다
- 에러 발생 시 복구 가이드가 제공됩니다

### 4. 자기 진화

프로젝트가 변하면 `/evolve`로 설정을 업데이트합니다. 사용 빈도는 `/stats`로 확인하여 미사용 항목을 정리할 수 있습니다.

### 5. 자연어 트리거

특별한 커맨드를 외우지 않아도 됩니다. "심층 분석", "보안 포함", "eco 모드", "끝까지" 같은 키워드를 포함하면 자동으로 적절한 모드가 활성화됩니다.

## 자주 쓰는 명령어 TOP 5

| 순위 | 명령 | 용도 |
|------|------|------|
| 1 | `/autopilot {작업}` | 복잡한 작업 자동 처리 |
| 2 | `/ralph {작업}` | 끝까지 완료 보장 |
| 3 | `/plan {작업}` | 구현 전 계획 세션 |
| 4 | `/review` | 코드 리뷰 |
| 5 | `/pr` | Pull Request 생성 |

## 자주 쓰는 자연어 패턴

```
"로그인 화면을 만들어줘"              → 기본 구현
"끝까지 결제 모듈 완성해"            → Ralph Mode (완료 보장)
"심층 분석해줘 인증 로직"            → deep-search 자동 활성화
"보안 포함해서 API 리팩토링"         → 보안 리뷰 자동 포함
"eco 모드로 이 버그 수정해줘"        → 토큰 절약 모드
"autopilot tdd로 인증 구현"          → 병렬 + TDD
도움말                               → 전체 기능 목록
```

## 문제가 생기면?

| 상황 | 해결 |
|------|------|
| 에이전트가 이상하게 동작 | `/doctor` — 설정 자기 진단 |
| setup을 안 했다는 메시지 | `/setup` — 프로젝트 초기화 |
| 의존성 추가 후 에이전트가 모름 | `/evolve` — 설정 진화 |
| 어떤 기능이 있는지 모르겠음 | `도움말` 또는 `/help` |
| 자율 루프가 멈추지 않음 | `취소` 또는 `/cancel` |
| 자율 루프가 중간에 끊김 | `/ralph` — 세션 시작 시 자동 감지되어 재개 안내 |
| 사용하지 않는 스킬이 많음 | `/stats` — 미사용 항목 확인 후 정리 |

## FAQ

Q: cursor_symbiote는 특정 언어/프레임워크 전용인가요?

A: 아닙니다. `/setup` 실행 시 코드베이스를 자동 분석하여 프로젝트 스택에 맞게 초기화됩니다. iOS, Web, Backend 등 어떤 프로젝트에서든 동작합니다.

---

Q: 모든 스킬을 외워야 하나요?

A: 아닙니다. Core 스킬 4개(code-accuracy, verify-loop, planning, git-commit)는 코드 작성 시 자동으로 참조됩니다. 나머지는 작업 맥락에 따라 Agent가 자동으로 로드합니다. `도움말`이라고 입력하면 전체 목록을 볼 수 있습니다.

---

Q: Ralph와 Autopilot의 차이는?

A: Ralph는 완료 보장에 초점(최대 10회 반복), Autopilot은 속도에 초점(병렬 처리, 최대 3회 반복)입니다. 간단히: 확실한 완료 → `/ralph`, 빠른 처리 → `/autopilot`.

---

Q: 위험한 명령을 실수로 실행할 수 있나요?

A: `guard-shell.sh` 훅이 `git push --force`, `rm -rf /` 등 위험 명령을 자동으로 차단합니다. 의도적으로 실행하려면 사용자가 명시적으로 승인해야 합니다.

---

Q: 팀 전체가 같은 설정을 사용할 수 있나요?

A: 네. `.cursor/` 폴더(kernel 룰, 스킬, 에이전트, 커맨드, 훅, 테스트)를 Git에 커밋하면 팀 전체가 공유할 수 있습니다. 런타임 상태(`project/state/`, `usage-data/`)만 `.gitignore`에 추가하면 됩니다.

---

Q: 기존 .cursor 설정이 있는 프로젝트에 적용하려면?

A: 기존 `.cursor`를 `.cursor.back`으로 이름 변경 후 새 `.cursor`를 복사하고 `/setup`을 실행합니다. setup이 자동으로 이전 설정을 분석하여 custom 파일을 이관합니다.
