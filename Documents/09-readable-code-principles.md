# 사람이 읽기 쉬운 코드 원칙

> 저자: JunyoungJung | 날짜: 2026-03-13

## 목적

cursor_symbiote는 AI Agent를 활용해 빠르게 작업하지만, 최종 산출물의 기준은 항상 사람이 이해할 수 있는 코드여야 합니다.
이 문서는 프로젝트 전반에 적용할 가독성 우선 원칙을 정의합니다.

## 핵심 원칙

1. 읽는 속도가 작성 속도보다 중요합니다.
2. 한 번에 하나의 의도만 드러내는 코드를 작성합니다.
3. 짧은 코드보다 분명한 코드를 우선합니다.
4. 영리한 축약보다 예측 가능한 구조를 선택합니다.
5. AI가 작성했는지 아닌지 드러나지 않을 만큼 자연스러운 코드만 허용합니다.

## 작성 기준

### 이름

- 변수, 함수, 타입 이름만 보고 역할이 추론되어야 합니다.
- 축약어는 팀이 이미 공유하는 경우에만 사용합니다.
- `data`, `temp`, `util`, `helper`, `manager`처럼 의미가 넓은 이름은 피합니다.
- boolean 값은 `is`, `has`, `can`, `should`로 시작해 질문 형태로 읽히게 만듭니다.

### 함수

- 함수는 하나의 책임만 가집니다.
- 함수 본문은 위에서 아래로 읽을 때 자연스럽게 이해되어야 합니다.
- 분기 수가 많아지면 조건을 설명하는 작은 함수로 분리합니다.
- 숨겨진 부작용이 있는 함수는 이름으로 그 사실을 드러냅니다.

### 구조

- 파일은 하나의 주제를 설명해야 합니다.
- 관련된 로직은 가까이 두고, unrelated 로직은 같은 파일에 섞지 않습니다.
- 계층 경계를 넘는 코드는 명시적인 인터페이스나 진입점을 통해 연결합니다.
- 공통화는 반복이 충분히 의미를 가질 때만 수행합니다.

### 주석

- 주석은 "무엇을 하는지"보다 "왜 이렇게 설계했는지"를 설명할 때만 작성합니다.
- 코드만 읽어도 알 수 있는 설명형 주석은 금지합니다.
- TODO, placeholder, 작업 흔적은 남기지 않습니다.

### 조건문과 흐름

- 예외 케이스는 초기에 반환해 본문을 평평하게 유지합니다.
- 조건식이 길어지면 의미 있는 이름의 predicate로 추출합니다.
- 중첩보다 조기 반환을 우선합니다.

### 데이터와 상수

- 매직 넘버와 매직 문자열은 이름 있는 상수로 올립니다.
- 데이터 구조는 사용하는 쪽의 읽기 비용이 낮아지도록 설계합니다.
- 외부 입력은 가능한 빨리 검증하고 정규화합니다.

## AI Agent 적용 규칙

- 에이전트는 기본적으로 가독성을 성능 미세 최적화보다 우선해야 합니다.
- 리팩터링 제안은 "줄 수 감소"가 아니라 "이해 비용 감소"를 기준으로 평가합니다.
- 자동 생성 코드는 제출 전 이름, 분기 구조, 주석 품질을 반드시 다시 점검합니다.
- 리뷰 단계에서는 동작 여부와 함께 "처음 읽는 사람이 이해 가능한가"를 별도 항목으로 확인합니다.

## 리뷰 체크리스트

- 이름만 읽어도 역할이 드러나는가
- 함수가 하나의 책임만 가지는가
- 불필요한 추상화나 조기 일반화가 없는가
- 주석 없이도 흐름이 이해되는가
- 예외 처리와 경계 조건이 눈에 잘 띄는가
- 다음 수정자가 안전하게 변경할 수 있는 구조인가

## 문서와 설정에도 동일하게 적용

이 원칙은 애플리케이션 코드에만 적용되지 않습니다.
cursor_symbiote의 Markdown 문서, 설정 파일, 훅 스크립트, 스킬 설명 역시 같은 기준으로 유지합니다.

- 제목은 문서의 목적이 바로 드러나야 합니다.
- 긴 문단보다 짧은 문단과 명확한 섹션을 우선합니다.
- 숫자, 구성 요소 수, 동작 설명은 문서 간 일관성을 유지합니다.
- 예시는 짧고 바로 실행 가능해야 합니다.
